Hiroo - Developer Guide

Hiroo is a web application that serves as a tool for a HR-specialilst. Let’s say you found 100 candidates that would fit into a company so you need to schedule a call (interview) with every single one of them one-on-one. This would take too much time and effort. HR-specialist come to Hiroo, creates an interview (or pick template) and use Hiroo as pre-selection tool. This way, HR specialist spend way less time just watching the results and AI can show some information to speed up the process even more.

Hiroo can also be used as a pre-selection tool for non-stop interview. For example, some companies are willing to hire a candidate whenever if he is good enough. Using Hiroo, HR specialist can prepare an interview (or pick a template) to check language or communication skills and only invite candidates to a personal interview if their score was at least 50%.

Terminology

  • HR is a type of account for HR specialist, a person that hires people into a company.
  • COMPANY is a type of account for a company leader, mostly used for reports and money deposits.
  • i18n is a localization system used in our projects. There is detailed information about how we ise i18n in our project in Localization section.
  • eteam, eteambuilding, eteambuilder and hiroom are all legacy names of the project while it was just and idea and prototype. These words can be found anywhere in project, and they all have the exact same meaning as hiroo.

Introduction

Hiroo consists of multiple projects and each of them may have its own specific documentation. If a project has a specific documentation, you can find it in projects/*.md.html file.

Frontend projects

AngularGlobal

Angular (frontend) project based on ThemeForest bootstrap theme originally used as both presentation and registration web. The New presentation web was created in a separate project so right now, AngularGlobal serves as a web to register and login new company. This project is served on register.hiroo.eu.

For more details, go to AngularGlobal Guide.

AngularClient

Angular (frontend) project based on ThemeForest bootstrap theme that contains presentation web focused on corporates and themed around COVID-19. This web was served on hiroo.eu but is now not used anywhere because Hiroo attempts to focus on small and medium-sized companies instead of corporates. Instead of updating information, we created a new project AngularCheckli to replace this one due to:

  • No light-theme support
  • Weak i18n support (not supporting our latest ecosystem)
  • Design not fitting for smaller companies (they like simple minimalistic web with decent animations)
  • Information not fitting for small companies

For more details, go to AngularClient Guide.

AngularCheckli

Angular (frontend) project based on Tailwild using TailwindUI components that contains new presentation web (used right now) focused on small and medium-sized companies. This web is served on hiroo.eu.

For more details, go to AngularCheckli Guide.

AngularInterviewRefactor

Angular (frontend) project based on ThemeForest bootstrap theme that contains interface for candidate. Originally, there was AngularInterview project, but it has been refactored and removed due to:

  • terrible code
  • terrible design
  • too many bugs

This project is served on interview.hiroo.eu.

AngularCompany

Angular (frontend) project based on ThemeForest bootstrap theme that contains interface for logged in company and hr. This is the biggest frontend application of the whole Hiroo project. This project is served on {company}.hiroo.eu where {company} is company domain provided when registering company. You cannot see this project unless you created company on register.hiroo.eu. When registering a company, there is the option to enter a domain name which has slug format. After finishing a registration and confirming company email address, you can visit your company Hiroo subdomain. For example, if my company is T-Systems and I used t-systems as a domain name, I can visit my company subdomain on t-systems.hiroo.eu. This is the website where everything is done (by both company and hr).

AngularPlatform

Angular (frontend) project based on Tailwind using TailwindUI components that is supposed to replace AngularCompany but is currently not finished (as of early 2021). We decided to recode this project due to:

  • Bad UI/UX (tables everywhere)
  • Bad dark/light theme support
  • Some responsivity problems
  • No i18n support

This project is in development but once it’s ready, it will replace AngularProject, so it will be served on {company}.hiroo.eu as explained in AngularCompany section

For more details, go to AngularPlatform Guide.

Backend projects

Server

This folder contains Node.JS project that serve as an API for the whole project. This project uses the latest localization system and can be found on api.hiroo.eu.
For more details, go to Server Guide.

SwaggerGUI

Project created to speed up the development of the API server. This folder includes Angular projects that provides GUI interface (on website) to define new routes. You simply enter basic information about the new route (route itself, what protection you need, what input you get, what output you give) and press copy button. This copies CLI script into your clipboard so you can just paste it into CLI inside Server folder and it will generate all documentation, guards, controllers and service wrapper for you. You just enter the service folder, find the generated method at top of the file and implement logic. Everything is generated and strictly typed, so you cannot send an incorrect response by mistake. You simply run npm run serve to start the website and it automatically connects to your API server on port 3000.

Email-Listener

This folder includes NodeJS project that does everything around mails. It uses MySQL connection to mail database that is filled by API server and whenever it sees new request, it sends the email. This email listener support multiple connections so it not only support emails from support@hiroo.eu, but also can send emails from specific division emails, for example tsystems.kosice@hiroo.eu.
Also, this server listens for new emails on division emails so it can automatically register candidate into an interview.

Interview-Server

Simple NodeJS script to provide interview functionality. It connects to MySQL database to see status of all interviews. Whenever the API changes some status, this server catches it and schedule the function to run specific actions in specific times. For example, it starts an interview (mark as started in database, schedule emails for candidates), remind candidates that the interview is about to finish and finally, mark the interview as finished. Long story short, this is simple script to manage scheduled actions. This script does not analyse interview results, the analysis function is inside Server the API folder.

InterviewTTSGenerator

Simple NodeJS script that takes input generated from AngularInterviewRefactor projects, parses it to get texts used by emoji and send it to Google API to generate text to speech (mp3). These MP3 files are then moved to AngularInterviewRefactor and used by the website. You only need to use this project if you change some static text said by emoji on AngularInterviewRefactor, you create new language or add new message said by emoji. To identify what messages are used by emoji in AngularInterviewRefactor, you can simply look into EmojiMp3Guard class (app/src/emoji-mp3.guard.ts)

Emoji-Convert

Small NodeJS script that converted purchased assets (mp4 video) of emoji animations into PNG assets. These assets were put into Server folder and are being used in AngularInterviewRefactor project. You won’t need to touch this script unless you want to modify emoji animations or add a new one.

Lang-Convert

This folder contains NodeJS script that was used by our deprecated language system to generate Google spreadsheet tables from translation files, so the translator can easily see the content. This script is no longer being used and was replaced by Localazy in our new localization system.

postleaf

Postleaf folder contains clone of the postleaf open-source project used for blog website that can be found on blog.hiroo.eu. This is pure clone with no changes and using their pre-defined theme. Be aware, this project may need Node V7, nor Node V12 depending on libraries updates. Please use NodeVersionManager to try both V7 and V12 when doing npm install. If one throws an error, please try the other one. So far, it always worked on V7.
