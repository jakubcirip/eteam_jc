AngularCheckli

This project is created using Angular framework. Feel free to look at Angular documentation to learn more. This is a small Angular app and has nothing special to it other than API class generation system and i18n system described below. Anyway, we strongly recommend to use components folder for components and pages folder for components that are using in Angular routing files to prevent mess in the project if it grows. To learn basics about this project, go to Developer guide document

Introduction

This project uses Angular 11, has the latest localization support and uses Tailwind 2 with it’s TailwundUI components.

Development

After cloning repository from GitHub, you enter the AngularCheckli project by typing 

$ cd AngularCheckli

To install all dependencies you will need for development, you run

$ npm i

We recommend you to serve this application on port 4203 because other ports may be used by different projects. To serve an application for development, you run 

$ npm run dev:en

After starting the development server, you can visit http://localhost:4203/ to see the application. All changes in src the folder will force the development server to recompile and refresh the website automatically.

This project uses our API generation system. You can find more information in API class generation document.

Localization

This project uses the latest localization system. To find a detail description about our localization system, please visit Localization document.

Deployment

To deploy the application, you run

$ npm run build

This will compile the website and create dist/twt a folder. You copy content of this folder into the static server directory on your server. Make sure to configure your static server to always return index.md.html in case of 404 error. Here is an example of such a configuration in nginx: 

server {
    listen 80;
    server_name hiroo.eu;
    root /var/www/hiroo.eu;

    location / {
        try_files $uri $uri/ /index.md.html =404;
    }
}

(this configuration does not include https. You can use certbot to update this config file and support https)
